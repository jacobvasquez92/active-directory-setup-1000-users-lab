Overview & Architecture

This lab builds a fully functional Active Directory environment with two VMs communicating over a private internal network — mirroring a real enterprise setup. The Domain Controller (DC) sits between the internet and the internal LAN, routing traffic for all client machines.

[ Internet ] | [ VMware NAT ] | ┌────────────────────────────────────┐ │ Domain Controller (DC) │ │ Windows Server 2022 │ │ ├─ NIC 1: NAT (internet) │ │ └─ NIC 2: Host-Only (LAN) │ │ IP: 172.16.0.1 │ │ Roles: AD DS · DNS · DHCP · NAT │ └────────────────────────────────────┘ | (internal host-only network) ┌────────────────────────────────────┐ │ CLIENT1 │ │ Windows 10/11 Enterprise │ │ NIC: Host-Only │ │ IP: 172.16.0.x (via DHCP) │ └────────────────────────────────────┘
🖥️

Domain Controller — Windows Server 2022 with AD DS, DNS, DHCP, and NAT

💻

Client Machine — Windows 10/11 joined to the domain

VM 1 — Domain Controller Setup

Create the VM in VMware

  1. 1
    Open VMware → click Create a New Virtual Machine
  2. 2
    Select "I will install the operating system later"
  3. 3
    Guest OS: Microsoft Windows → Windows Server 2022
  4. 4
    Name the VM: DC (or Domain-Controller)
  5. 5
    Disk size: 60 GB → Store as a single file
  6. 6
    Click "Customize Hardware" before finishing — configure settings below
💡
Do NOT use "Installer disc image file" at the VM creation step — it can trigger easy-install mode and cause boot errors. Attach the ISO manually after the VM is created.

VMware Hardware Settings — Domain Controller

SettingValue
Memory2–4 GB
Processors1 CPU, 2 Cores
Network Adapter 1NAT — connects to internet
Network Adapter 2Host-Only — internal LAN (add via "Add...")
CD/DVD DriveBrowse → Windows Server 2022 ISO
🔄
VMware vs VirtualBox: VirtualBox uses an "Internal Network" adapter for isolated LAN traffic. In VMware, Host-Only achieves the same effect — VMs can talk to each other but not the external network directly.

Install Windows Server 2022

  1. 1
    Power on the VM
  2. 2
    Act fast! When you see "Press any key to boot from CD/DVD..." — press immediately. You have ~3 seconds.
  3. 3
    In the setup wizard, select Windows Server 2022 Standard Evaluation (Desktop Experience)
  4. 4
    Choose Custom Install → select the unallocated disk → Next
  5. 5
    Let the install complete, then set a strong Administrator password
⚠️
You must choose Desktop Experience — the non-GUI (Core) version has no desktop, icons, or taskbar and is not suitable for this lab.

Install VMware Tools

VMware Tools enables proper display scaling, clipboard sharing, and drag-and-drop between host and VM.

  1. 1
    Log into the VM as Administrator
  2. 2
    VMware menu bar → VM → Install VMware Tools...
  3. 3
    Inside the VM, open File Explorer → D: drive → run setup64.exe
  4. 4
    Follow the wizard (choose Typical) → Restart the VM
04

Configure Network Adapters

The DC has two NICs. We need to identify, rename, and assign a static IP to the internal one.

Identify Each Adapter

  1. 1
    Open Network & Internet Settings → Change adapter options
  2. 2
    Double-click each adapter → Details → look at the IP address
  3. 3
    IP starting with 169.254.x.x → this is INTERNAL (Host-Only, no DHCP yet)
    IP starting with 192.168.x.x → this is INTERNET (NAT)
  4. 4
    Rename them: Ethernet0 → INTERNET  |  Ethernet1 → INTERNAL

Assign Static IP to INTERNAL Adapter

  1. 1
    Double-click INTERNAL → Properties
  2. 2
    Select Internet Protocol Version 4 (TCP/IPv4) → Properties
  3. 3
    Enter the values shown below → click OK
IP Config — INTERNAL Adapter
IP Address:      172.16.0.1
Subnet Mask:     255.255.255.0
Default Gateway: (leave blank — NAT adapter handles routing)

Preferred DNS:   127.0.0.1   ← DC uses itself as DNS server
Alternate DNS:   8.8.8.8
💡
Leave the INTERNET adapter alone — it gets its IP automatically from VMware's NAT DHCP service.

06

Create a Domain Admin Account

Using the built-in Administrator account directly is poor practice. Create a dedicated admin account instead.

Create an Organizational Unit (OU)

OUs are AD containers — like folders — that Group Policies can be applied to. We'll create one for admin accounts.

  1. 1
    Start → Windows Administrative Tools → Active Directory Users and Computers (pin this to the taskbar)
  2. 2
    Right-click your domain (mydomain.com) → New → Organizational Unit
  3. 3
    Name it _ADMINS → OK

Create the Admin User

  1. 1
    Right-click _ADMINS → New → User
  2. 2
    User logon name: a-yourname (convention: a- prefix for admins)
  3. 3
    Set a password → check "Password never expires" (lab only) → Finish

Grant Domain Admin Privileges

  1. 1
    Right-click the new user → Properties
  2. 2
    Member Of tab → Add...
  3. 3
    Type Domain Admins → Check Names → OK → Apply
  4. 4
    Sign out → click Other user → log in as a-yourname
07

Install RAS / NAT

Remote Access Service (RAS) with NAT allows CLIENT1 — which only has a Host-Only adapter — to reach the internet through the Domain Controller.

Add the Remote Access Role

  1. 1
    Server Manager → Add Roles and Features
  2. 2
    Server Roles: check Remote Access → Next
  3. 3
    Role Services: check Routing → Add Features → Next → Install

Enable and Configure NAT

  1. 1
    Server Manager → Tools → Routing and Remote Access
  2. 2
    Right-click your server → Configure and Enable Routing and Remote Access
  3. 3
    Configuration: select Network Address Translation (NAT)
  4. 4
    Network Interface: select the INTERNET adapter → Finish
✅
The server icon should now show a green upward arrow. NAT is active and routing is enabled.

Configure DHCP Server

The DHCP server will automatically assign IP addresses (172.16.0.100–200) to clients on the internal network.

Add the DHCP Role

  1. 1
    Server Manager → Add Roles and Features
  2. 2
    Check DHCP Server → Add Features → Install

Configure the DHCP Scope

  1. 1
    Server Manager → Tools → DHCP
  2. 2
    Expand your server → right-click IPv4 → New Scope...
FieldValue
Scope Name172.16.0.100-200
Start IP172.16.0.100
End IP172.16.0.200
Subnet Mask255.255.255.0 (Length: 24)
Default Gateway172.16.0.1 — the DC's internal IP
DNS ServerLeave default (configured by AD)
  1. 3
    "Configure DHCP options now?" → Yes → set Default Gateway to 172.16.0.1
  2. 4
    Skip WINS → "Yes, activate this scope now" → Finish
  3. 5
    In DHCP console → right-click server name → Authorize → right-click → Refresh
✅
Both IPv4 and IPv6 should now show a green checkmark. DHCP is active.

Bulk Create Users with PowerShell

We'll auto-generate ~1,000 user accounts using Josh Madakor's PowerShell script — simulating a real enterprise directory.

Prepare the Environment

  1. 1
    Server Manager → Configure this local server
  2. 2
    Find IE Enhanced Security Configuration → turn Off for both Admins and Users

Download & Customize the Script

  1. 1
    Browse to: github.com/joshmadakor1/AD_PS ↗
  2. 2
    Click Code → Download ZIP → extract the folder
  3. 3
    Open names.txt → add your own name at the top → save
📝
Adding your name ensures you have a personal account to log into CLIENT1 with later.

Run the Script

  1. 1
    Open Windows PowerShell ISE as Administrator
  2. 2
    Open 1_CREATE_USERS.ps1 from the extracted folder
  3. 3
    In the terminal pane, run:
PowerShell 
Set-ExecutionPolicy Unrestricted
# Select "Yes to All" when prompted
  1. 4
    Navigate to the script directory:
PowerShell 
cd C:\Users\a-yourname\Downloads\AD_PS-master\AD_PS-master
  1. 5
    Press F5 (or click the green Run button) — watch users being created in real time
  2. 6
    Verify: open Active Directory Users and Computers → check the _USERS OU
🔑
All auto-created accounts use the default password: Password1

Troubleshooting

VM won't boot from ISO
Go to VM Settings → CD/DVD Drive → confirm the ISO path is set and "Connect at power on" is checked. Also verify the ISO file isn't corrupted.
Missed the "press any key" prompt
In VMware: VM Settings → Options → Boot → set Power-on boot delay to 5000 ms. This gives you 5 seconds to press a key at the boot screen.
CLIENT1 has no IP address
Ensure the DHCP scope is authorized (green checkmark in DHCP console). Verify CLIENT1's NIC is set to Host-Only and that it matches the DC's Host-Only network segment.
Can't ping google.com from CLIENT1
Verify NAT is enabled on the DC (green arrow in Routing and Remote Access). Check the INTERNAL adapter on the DC is 172.16.0.1. Confirm the DHCP scope default gateway is set to 172.16.0.1.
Domain join fails
On CLIENT1, run ipconfig /all — DNS Server should show 172.16.0.1. If not, set it manually in adapter properties. Also confirm the DC is running and the domain name is typed correctly.
PowerShell script won't run
Open PowerShell ISE as Administrator. Run Set-ExecutionPolicy Unrestricted and choose Yes to All. Then navigate to the correct directory before running the script.
